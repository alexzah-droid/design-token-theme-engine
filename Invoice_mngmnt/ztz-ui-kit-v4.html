Layout & Grid

Breakpoints · Колонки · Margins · Gutters

Система построена на трёх брейкпоинтах и гибкой колоночной сетке. На mobile — 4 колонки, на tablet — 8, на desktop — 12. Отступы кратны 4 pt; gutters растут с экраном. Справа сверху — симулятор вьюпорта, позволяет проверить поведение любого блока на разных размерах.

Fluid vs Adaptive

Fluid · плавный

Растягивается непрерывно

Используем clamp() и проценты. Подходит для типографики, размеров hero-блоков, отступов.

font-size: clamp(34px, 6vw, 56px);
width: min(1280px, 100% - 48px);
Adaptive · ступенчатый

Меняется на брейкпоинтах

Через @media. Подходит для структуры: количества колонок, смены nav-bar на гамбургер, скрытия секций.

@media (max-width: 720px){
  .grid{grid-template-columns: 1fr}
  nav{display: none}
}

Mobile-specifics

Touch targets

Минимум 44 × 44 px по WCAG 2.5.5. Зазор между соседними таргетами — не меньше 8 px.

Safe area

Учитываем вырезы и панели жестов через env(safe-area-inset-*). Sticky-элементы — в безопасной зоне.

Горизонтальный скролл

Широкие блоки (таблицы, гант, графики) — в контейнере с overflow-x:auto. Основной поток — без гор. скролла.

07

Motion Design

Durations · Easing · Свойства

Анимация — инструмент ясности, а не украшение. Три базовых длительности и четыре easing-кривые покрывают все случаи. Анимируем только GPU-дружественные свойства. Всегда уважаем prefers-reduced-motion.

Durations

Fast · 150 ms--dur-fast · hover, color, opacity
Medium · 300 ms--dur-med · dropdown, accordion, tabs
Slow · 500 ms--dur-slow · modal, drawer, page transitions

Easing-кривые

Standardcubic-bezier(.2,.8,.2,1) По умолчанию для входящих изменений, раскрытий, фокус-рингов.
Ease-outcubic-bezier(0,0,.2,1) Элементы въезжают в экран, fade-in, growth.
Ease-incubic-bezier(.4,0,1,1) Элементы уходят из экрана, fade-out, dismiss.
In-outcubic-bezier(.4,0,.2,1) Перемещения внутри экрана (drawer, tab switch).

Анимируемые свойства

✓ Можно · GPU-дружественные

  • transformscale, rotate, translate
  • opacity0 → 1, fade-in/out
  • filterblur, brightness
  • colorтекст и иконки
  • background-colorфон карточек
  • border-colorfocus, error
  • box-shadowhover-lift
  • max-heightaccordion (осторожно)

× Нельзя · вызывают reflow

  • width / heightвместо этого — transform: scale
  • top / leftвместо — translate
  • margin / paddingраняет layout
  • font-sizeрефлоу всего текста

Micro-interactions · живые примеры

Loader

300 ms · linear · transform: rotate

Pulse · attention

1500 ms · ease-out · scale + opacity

Hover-lift

150 ms · standard · transform + shadow

Правило: все переходы — через CSS-переменные --dur-1…--dur-4 и --ease-out. Никогда не пишем жёсткие значения в компонентах.

Доступность · A11y

WCAG 2.2 AA · EN 301 549 · ГОСТ Р 52872

Доступность — это не «галочка для отчёта», а часть контракта качества. Корпоративные системы используются ежедневно сотнями людей, в том числе с нарушениями зрения, моторики и когнитивных функций. UI Kit ЗТЗ соответствует трём международным и национальным стандартам.

Стандарты соответствия

WCAG 2.2 · Level AA
Web Content Accessibility Guidelines

Базовый международный стандарт W3C. 4 принципа: воспринимаемость, управляемость, понятность, надёжность.

EN 301 549 · v3.2.1
Европейский стандарт

Требования к ИКТ-продуктам в госсекторе ЕС. Базируется на WCAG 2.1 AA + аппаратные требования.

ГОСТ Р 52872-2019
Российский стандарт

«Интернет-ресурсы и другая информация… Требования доступности для инвалидов по зрению».

Section 508 · Refresh
США / федеральный

Требования к ИКТ для федеральных агентств. Гармонизирован с WCAG 2.0 AA.

Контрастность · WCAG 1.4.3 / 1.4.11

Все основные цветовые пары проверены на коэффициент контрастности. Минимум для обычного текста — 4.5 : 1, для крупного (≥18 pt / 14 pt bold) и UI-компонентов — 3 : 1.

Клавиатурная навигация · WCAG 2.1.1 / 2.4.7

Весь интерфейс управляем без мыши. Фокус видимый — золотая обводка 3px rgba(176,138,62,.35) через :focus-visible.

Все компоненты имеют корректную семантику и ARIA-атрибуты. Тестировались с NVDA (Win), JAWS 2024 (Win), VoiceOver (macOS / iOS), TalkBack (Android).

Чек-лист соответствия · WCAG 2.2 AA

  1. 1.1.1 · Non-text ContentАльтернативный текст для всего нетекстовогоИконки без подписи имеют aria-label; декоративные — aria-hidden="true"; графики SVG — role="img" + описание.
  2. 1.3.1 · Info and RelationshipsСемантическая разметкаИспользуем <header>, <nav>, <main>, <section>; заголовки H1–H3 в строгой иерархии; формы — через <label for>.
  3. 1.4.3 · Contrast (Minimum)Контраст текста ≥ 4.5 : 1Все цветовые пары проверены инструментами axe-core и WebAIM Contrast Checker. Светлая и тёмная темы — независимо.
  4. 1.4.4 · Resize TextМасштабирование текста до 200%Шрифты — в rem, контейнеры fluid. Без потери функциональности на 200% zoom.
  5. 1.4.10 · ReflowАдаптация при 320 × 256 pxБез горизонтального скролла на основном контенте. Таблицы, гант и широкие графики — в управляемом scrollable-контейнере.
  6. 1.4.11 · Non-text ContrastКонтраст UI-элементов ≥ 3 : 1Границы полей, фокус-ринг, иконки и графики — все ≥ 3 : 1 относительно фона в обеих темах.
  7. 2.1.1 · KeyboardПолный доступ с клавиатурыВсе интерактивные элементы достижимы и активируемы через Tab, Enter, Space, стрелки, Esc. Без ловушек фокуса.
  8. 2.4.1 · Bypass BlocksSkip-link «к основному содержанию»Первый focusable-элемент на странице — ссылка .skip-link, ведущая на #main.
  9. 2.4.7 · Focus VisibleВидимый индикатор фокусаЗолотая обводка 3 px при :focus-visible на всех элементах. Контраст ≥ 3 : 1.
  10. 2.5.5 · Target SizeHit-area ≥ 44 × 44 pxВсе кнопки, табы, чипы и иконки-кнопки имеют физический размер не меньше 44 × 44 px (или 24 × 24 с зазором 24 px по 2.5.8).
  11. 2.3.3 · Animation from InteractionsУважение prefers-reduced-motionВсе анимации (раскрытия, переходы, скелетоны) отключаются при системной настройке «уменьшить движение».
  12. 3.3.1 · Error IdentificationПонятные сообщения об ошибкахПоля в состоянии ошибки имеют aria-invalid="true" + связанный aria-describedby с описанием.
  13. 4.1.2 · Name, Role, ValueКорректные ARIA-роли и значенияВсе кастомные контролы (dropdown, accordion, switch, tabs) имеют правильные роли и состояния, синхронизированные с DOM.
  14. 4.1.3 · Status MessagesLive-регионы для динамических обновленийУведомления, прогресс-бары и счётчики обновляются через role="status" / role="alert".

Дополнительные практики

Управление фокусом

  • Модалки: focus-trap внутри + возврат на триггер при закрытии
  • Дропдауны: фокус на первый option при открытии
  • Маршрутизация: фокус на H1 нового экрана при SPA-переходе
  • Skip-link виден сразу при Tab с любого места

Когнитивная доступность

  • Понятные подписи кнопок («Отправить заявку», не «OK»)
  • Иерархия заголовков без пропусков уровней
  • Время сессий не лимитировано (или предупреждение + продление)
  • Никогда только цвет: статусы дополняются иконкой и текстом

Регламент проверки: автотесты axe-core в CI на каждом PR; ручное тестирование с NVDA + VoiceOver — раз в спринт; внешний аудит — раз в полгода.

11

Frontend-гайд

Подключение и правила

Тон и голос бренда

Говорим спокойно, по-инженерному. Конкретно, без маркетингового пафоса. Мы — завод, и это видно и слышно: цифры и артикулы — моноширинно, заголовки — серифом, тело — гротеском.

Темы и токены

Все цвета, отступы и тени — через CSS-переменные в assets/tokens.css. Темы переключаются атрибутом data-theme="light|dark" на <html>. Режим auto уважает prefers-color-scheme; состояние сохраняется в localStorage.

Анимации

Стандартная длительность — 180–260 мс, easing — cubic-bezier(.2,.8,.2,1). Раскрытия (dropdown, accordion) анимируют max-height+opacity+translateY(-4px). Всегда уважаем prefers-reduced-motion: reduce.

Адаптивность

Mobile-first. Брейкпоинты: 720px (mobile→tablet), 1024px (tablet→desktop). Сетка перестраивается на 1fr на мобильных; широкие графики, таблица и гант — в горизонтально прокручиваемых контейнерах. Hero уплотняется, навигация сворачивается в гамбургер.
