# Claude Code Starter v5.0.0

## Summary

`v5.0.0` moves the public `claude-code-starter` repository from the old `v4` framework layout to the new Claude Code native operating model.

This release restores the simple installer UX from earlier versions:
- one public `init-project.sh` at repo root;
- one launcher for new, existing, legacy, and partial-install projects;
- one release payload archive expected by the launcher.

At the same time, `v4` is not discarded. The old framework tree is preserved in-repo under `archive/v4-working-tree/`, and the pre-migration head is tagged as `archive-v4-head-2026-04-06`.

## Included Release Assets

Expected GitHub Release assets for `v5.0.0`:
- `init-project.sh`
- `framework.tar.gz`
- `checksums.txt`
- `RELEASE_NOTES.md`

## Operational Changes

### Public installer
- root `init-project.sh` is the only public installer entrypoint
- standalone mode downloads the payload automatically
- fallback chain:
  - release archive
  - git clone
  - repository snapshot

### Claude Code native primitives
- `rules` replace large operational prose
- `skills` replace command/protocol sprawl
- `agents` formalize delegatable roles
- `hooks` provide background guardrails

### Framework state model
- `repo_access` is now explicit in `manifest.md`
- `private-solo` allows framework state in git
- `private-shared` and `public` keep framework state local
- `switch-repo-access.sh` handles safe mode transition

## Upgrade Notes

If you are coming from `v4`:
- the old runtime is archived, not removed
- the root repository now represents `v5`
- installer expectations have changed from `v4` release artifacts to the new `framework.tar.gz` payload

If you are installing into a host project:
- copy or download `init-project.sh`
- run it at the project root
- switch `repo_access` before the first framework commit if the host repo is shared/public

## Known Limitations

- `migrate.sh` still depends on `python3` for safe JSON merge of `.claude/settings.json`
- if framework state already exists in upstream shared/public history, mode switching still requires history cleanup or a fresh branch
- release publishing is still a manual GitHub Release step, even though asset generation is scripted
